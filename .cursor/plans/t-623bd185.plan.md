## Phase 1: Project Setup and Core Infrastructure

### 1.1 Backend Foundation

- Initialize Express server with middleware (cors, helmet, morgan)
- Configure MongoDB connection with Mongoose
- Set up Socket.io for real-time communication
- Create base error handling and response utilities
- Configure Multer for local image uploads to `/uploads` folder

### 1.2 Frontend Foundation

- Initialize Expo project with `expo-router`
- Configure Gluestack UI theming (custom colors, typography)
- Set up Zustand stores (auth, notifications)
- Configure React Query for API data fetching
- Create API service layer with axios/fetch

---

## Phase 2: Authentication System

### 2.1 Backend Auth

**Models:** `User` schema with role enum (customer, tailor, admin)

**Endpoints:**

- `POST /api/auth/register` - Create account with role
- `POST /api/auth/login` - JWT token generation
- `GET /api/auth/me` - Get current user
- `POST /api/auth/logout` - Invalidate token

**Key files:** `models/User.js`, `controllers/authController.js`, `middleware/auth.js`

### 2.2 Frontend Auth

- Login/Register screens with form validation (react-hook-form + zod)
- Secure token storage (expo-secure-store)
- Auth context with auto-login on app start
- Role-based navigation (customer vs tailor vs admin views)

---

## Phase 3: Tailor Profiles and Portfolio

### 3.1 Backend

**Models:** `Tailor` schema (embedded fabrics array, portfolio array)

**Endpoints:**

- `GET /api/tailors` - List with filters (location, rating, availability)
- `GET /api/tailors/:id` - Profile details
- `PUT /api/tailors/:id` - Update profile (tailor only)
- `POST /api/tailors/portfolio` - Add portfolio item with image
- `DELETE /api/tailors/portfolio/:id` - Remove portfolio item
- `GET /api/search/tailors` - Advanced search with filters

### 3.2 Frontend Screens

- **Customer:** Home (tailor listings), Search/Filter, Tailor Profile, Portfolio Gallery
- **Tailor:** Dashboard, Profile Edit, Portfolio Management, Fabric Management

**Chart Implementation (Tailor Dashboard):**

- Orders Over Time (Line Chart) - Last 7/30 days order trends
- Order Status Distribution (Donut Chart) - Visual breakdown of order statuses
- Earnings Trend (Area Chart) - Optional, revenue growth visualization
- Use react-native-gifted-charts library
- Chart data from API: `/api/tailors/dashboard?period=7d|30d`

---

## Phase 4: Order System

### 4.1 Backend

**Models:** `Order` schema with status history, measurements by type

**Endpoints:**

- `POST /api/orders` - Create order
- `GET /api/orders` - List orders (filtered by role)
- `GET /api/orders/:id` - Order details
- `PATCH /api/orders/:id/status` - Update status (tailor only)

**Business Logic:**

- Auto-generate order number
- Calculate total (fabric + express fee)
- Dynamic measurements based on stitchingType (Men/Women/Children)

### 4.2 Frontend Screens

- **Customer:** Place Order Form (multi-step), Order Confirmation, My Orders, Order Detail
- **Tailor:** Order List (filterable), Order Detail, Update Status modal

---

## Phase 5: Real-time Chat

### 5.1 Backend Socket.io

**Models:** `Conversation`, `Message`

**Socket Events:**

- `join_conversation` - User joins room
- `send_message` - Emit new message
- `message_received` - Broadcast to participants
- `typing` / `stop_typing` - Typing indicators
- `mark_read` - Read receipts

**REST Endpoints:**

- `GET /api/conversations` - User's conversations
- `GET /api/conversations/:id/messages` - Message history
- `POST /api/messages` - Send message (fallback)

### 5.2 Frontend

- Chat List screen with last message preview
- Chat Screen with real-time updates
- Image sharing via image picker
- Typing indicators and read receipts

---

## Phase 6: Reviews and Ratings

### 6.1 Backend

**Model:** `Review` schema

**Endpoints:**

- `POST /api/reviews` - Create review (completed orders only)
- `GET /api/reviews/tailor/:id` - Tailor's reviews

**Business Logic:**

- One review per order validation
- Auto-update tailor rating on new review
- 7-day edit window enforcement

### 6.2 Frontend

- Write Review screen (accessed from completed order)
- Reviews section on Tailor Profile
- Star rating component

**Endpoints:**

## Key Implementation Notes

1. **Image Uploads:** Multer saves to `/backend/uploads/`, served statically via Express
2. **Auth Middleware:** JWT verification on protected routes, role checking for admin endpoints
3. **Socket.io Auth:** Token verification on connection, room-based messaging
4. **Measurements:** Dynamic form fields based on stitchingType selection
5. **Search:** MongoDB text indexes on tailor name, businessName, location fields
6. **Charts:** Use react-native-gifted-charts for data visualization

   - Works perfectly with Expo (no native code)
   - Supports Line, Bar, Pie, Area, Donut chart types
   - Styling matches app theme (#C9A227 gold, #1D3A5F dark blue)
   - Charts wrapped in Card components with white background
   - API endpoints should return time-series data for charts

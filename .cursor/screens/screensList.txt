# TailorConnect - Detailed Screen Specifications

This document provides comprehensive specifications for all screens in the TailorConnect mobile app. Each screen includes detailed field definitions, UI components, validation rules, API endpoints, and navigation patterns.

---

## Design System Reference

**Color Scheme:**
- Primary Gold: #C9A227
- Dark Blue: #1D3A5F
- Background: #F5F6F8
- White: #FFFFFF
- Error Red: #EF4444

**Common Components:**
- LabeledInputField: Input with label, icon, error handling
- GradientButton: Primary action button with gold gradient
- SafeAreaView: Screen container
- ScrollView: Scrollable content container
- VStack: Vertical stack layout

## Navigation Structure

### Navigation Strategy

The app uses a **dual navigation system**:
1. **Bottom Tab Navigation** - For primary/frequently accessed screens (5 tabs max)
2. **Sidebar/Drawer Navigation** - For secondary actions, settings, and profile management

### Why Both?

- **Bottom Tabs**: Quick access to core features, always visible, mobile-friendly UX
- **Sidebar/Drawer**: Access to less frequent but important features, keeps UI clean and organized

### Bottom Tab Navigation

**Library:** expo-router Tabs component

**Styling:**
- Active tab: #C9A227 (gold)
- Inactive tab: Gray (#9CA3AF)
- Background: #FFFFFF
- Border top: Light gray (#E5E7EB), 1px
- Tab bar height: 60-70px
- Icon size: 24px
- Label font size: 12px
- Active label: Bold, #C9A227
- Inactive label: Regular, #9CA3AF

**Icons:** MaterialCommunityIcons
**Badges:** Red circle (#EF4444) with white number, positioned top-right of icon
**Max tabs:** 5 (optimal for mobile screens)

### Sidebar/Drawer Navigation

**Library:** expo-router Drawer or custom drawer component

**Styling:**
- Background: #FFFFFF
- Header background: Gradient (#C9A227 to #A88B1E) or solid #1D3A5F
- Header text: White (#FFFFFF)
- Menu item text: #1D3A5F, 16px
- Active item: #C9A227 background (light) or left border (4px)
- Divider: Light gray (#E5E7EB), 1px
- Logout: Red (#EF4444)
- Item padding: 16px vertical, 24px horizontal
- Icon size: 24px
- Icon color: #1D3A5F (active: #C9A227)

**Trigger:** Hamburger menu icon (MaterialCommunityIcons "menu") in header top-left
**Width:** 280-300px
**Animation:** Slide from left with smooth transition

### Header Component (Common to All Screens)

**Layout:**
- SafeAreaView at top
- Height: 56-60px
- Background: #FFFFFF
- Border bottom: Light gray (#E5E7EB), 1px

**Left Section:**
- Hamburger menu icon (opens drawer)

## Phase 2: Authentication System

### 1. Login Screen
**Purpose:** User authentication with email and password

**Layout:**
- SafeAreaView (backgroundColor: #F5F6F8)
- ScrollView (keyboardShouldPersistTaps="handled")
- VStack (centered, paddingHorizontal: 24)
- Logo container (140x140, centered)
- Heading: "Welcome Back!" (28px, bold, #1D3A5F)

**Form Fields:**
1. **Email Address**
   - Component: LabeledInputField
   - Type: email
   - Icon: MaterialCommunityIcons "email-lock-outline" (#C9A227)
   - Validation: Required, valid email format
   - Keyboard: email-address
   - AutoCapitalize: none

2. **Password**
   - Component: LabeledInputField
   - Type: password
   - Icon: Eye/EyeOff (lucide-react-native, toggle visibility)
   - Validation: Required, min 6 characters
   - SecureTextEntry: true (toggleable)

**Actions:**
- Forgot Password Link (TouchableOpacity, right-aligned, #1D3A5F)
- Login Button (GradientButton, full width, height: 54, borderRadius: 27)
- Register Link (TouchableOpacity, centered, "Don't have an account? Register")

**Navigation:**
- On success: router.replace('/(tabs)') based on user role
- On register press: router.push('/register')
- On forgot password: router.push('/forgot-password') (future)

**API Endpoint:**
- POST /api/auth/login
- Request: { email, password }
- Response: { token, user: { id, name, email, role } }

**Validation Schema (zod):**
```typescript
{
  email: z.string().min(1, 'Email is required').email('Invalid email'),
  password: z.string().min(1, 'Password is required').min(6, 'Min 6 characters')
}
```

### 2. Register Screen
**Purpose:** Account creation with role selection (customer/tailor)

**Layout:**
- SafeAreaView (backgroundColor: #F5F6F8)
- ScrollView (keyboardShouldPersistTaps="handled")
- VStack (centered, paddingHorizontal: 24)
- Logo container (140x140, centered)
- Heading: "Create Account" (28px, bold, #1D3A5F)

**Form Fields:**
1. **User Type Selection**
   - Component: UserTypeToggle
   - Options: "Customer" | "Tailor"
   - Default: "customer"
   - Validation: Required

2. **Username**
   - Component: LabeledInputField
   - Type: text
   - Icon: MaterialCommunityIcons "account-outline" (#C9A227)
   - Validation: Required, min 3 characters
   - AutoCapitalize: none

3. **Password**
   - Component: LabeledInputField
   - Type: password
   - Icon: Eye/EyeOff (toggle visibility)
   - Validation: Required, min 6 characters
   - SecureTextEntry: true (toggleable)

4. **Confirm Password**
   - Component: LabeledInputField
   - Type: password
   - Icon: Eye/EyeOff (toggle visibility)
   - Validation: Required, must match password
   - SecureTextEntry: true (toggleable)

**Actions:**
- Register Button (GradientButton, full width, height: 54, borderRadius: 27)
- Login Link (TouchableOpacity, centered, "Already have an account? Login")

**Navigation:**
- On success: router.replace('/(tabs)') based on selected role
- On login press: router.push('/login')

**API Endpoint:**
- POST /api/auth/register
- Request: { username, email, password, role: 'customer' | 'tailor' }
- Response: { token, user: { id, name, email, role } }

**Validation Schema (zod):**
```typescript
{
  userType: z.enum(['customer', 'tailor'], { required_error: 'Select user type' }),
  username: z.string().min(1, 'Required').min(3, 'Min 3 characters'),
  password: z.string().min(1, 'Required').min(6, 'Min 6 characters'),
  confirmPassword: z.string().min(1, 'Required')
}.refine(data => data.password === data.confirmPassword, {
  message: "Passwords don't match",
  path: ['confirmPassword']
})
```

### 3. Home Screen (Customer)
**Purpose:** Browse and discover available tailors

**Layout:**
- SafeAreaView (backgroundColor: #F5F6F8)
- ScrollView or FlatList
- Header with search bar
- Tailor cards in grid/list view

**Components:**

   - Component: LabeledInputField (without label, search icon)
   - Icon: MaterialCommunityIcons "magnify" (#C9A227)
   - Placeholder: "Search tailors..."
   - Debounced search (300ms)

2. **Filter Button**
   - Component: TouchableOpacity with icon
   - Icon: MaterialCommunityIcons "filter-variant" (#C9A227)
   - Navigate to Search/Filter Screen

3. **Tailor Card**
   - Profile Image (circular, 80x80)
   - Business Name (Heading, 18px, bold)
   - Location (Text with icon, MaterialCommunityIcons "map-marker")
   - Rating (Star component, 0-5 stars)
   - Specializations (Chip/Tag components, max 3)
   - Availability Badge (green dot + "Available" text)
   - "View Profile" Button (GradientButton, small)

**Actions:**
- Tap card: Navigate to Tailor Profile Screen
- Pull to refresh: Reload tailors list
- Infinite scroll: Load more tailors

**API Endpoint:**
- GET /api/tailors?page=1&limit=20&location=&rating=&availability=true
- Response: { tailors: [{ id, businessName, location, rating, profileImage, specializations, isAvailable }], total, page }

**Data Structure:**
```typescript
{
  id: string;
  businessName: string;
  location: string;
  rating: number; // 0-5
  profileImage?: string;
  specializations: string[];
  isAvailable: boolean;
  userId: string;
}
```

## Implementation Notes

### Common Patterns
1. **Form Validation**: Always use react-hook-form + zod
2. **Loading States**: Show loading spinner on GradientButton during API calls
3. **Error Handling**: Display error messages below form fields or as toast
4. **Image Upload**: Use expo-image-picker with ActionSheet for options
5. **Navigation**: Use expo-router for all navigation
6. **Real-time Updates**: Use Socket.io client for chat and notifications
7. **Data Fetching**: Use @tanstack/react-query for API calls
8. **State Management**: Use Zustand for global state (auth, notifications)

### Color Usage
- Primary actions: GradientButton (gold gradient)
- Secondary actions: Outlined buttons (#C9A227 border)
- Destructive actions: Red buttons (#EF4444)
- Text: #1D3A5F for primary, gray for secondary
- Background: #F5F6F8 for screens, #FFFFFF for cards

### Icon Library
- MaterialCommunityIcons for most icons
- lucide-react-native for simple icons (Eye, EyeOff, etc.)

### Chart Library
- **react-native-gifted-charts**: Primary charting library
  - Installation: `npm install react-native-gifted-charts react-native-svg`
  - Supports: Line, Bar, Pie, Area, Donut, Stacked Bar charts
  - Works perfectly with Expo (no native code required)
  - TypeScript support included
  - Customizable styling to match app theme
  - Chart components should be wrapped in Card components
  - Use app color scheme: #C9A227 (gold), #1D3A5F (dark blue), status colors
  - Chart cards: White background (#FFFFFF), rounded corners, shadow, padding: 16px

### Responsive Design
- Use percentage widths for flexible layouts
- Max width: 400px for forms (centered)
- Padding: 24px horizontal on screens
- Card padding: 16px

- Handles auth state
- Routes to appropriate role-based navigation
- Contains GluestackUIProvider
